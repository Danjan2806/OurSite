2. **Обязательные фичи (с работающим кодом):**
- **Шаг 1 — выбор типа бизнеса**:
  - `LANDING` (стандартные SEO-блоки, форма захвата)
  - `ECOMMERCE` (корзина, каталог, платежи mock)
  - `MUSIC_LABEL` (личный кабинет артиста, загрузка треков, плеер)
  - `FITNESS` (расписание занятий, запись, абонементы)
- **Динамическое создание БД**: эндпоинт `POST /api/tenants` создаёт новую PostgreSQL БД через `CREATE DATABASE tenant_{uuid}` и выполняет миграцию с нужными таблицами под тип бизнеса.
- **Конструктор блоков**:
  - Блоки: Hero, Features, Pricing, Contacts, ProductGrid (для ecom), Schedule (fitness), MusicPlayer (лейбл)
  - Перетаскивание с левой панели на канвас
  - Редактирование текста/цветов прямо на канвасе (contenteditable или inline-редактор)
- **Live-превью**: через WebSocket обновляется iframe с рендером текущего макета.
- **Публикация**: генерирует статический Next.js сайт, заливает в S3-совместимое хранилище (в коде сделай имитацию через локальную папку) и выдаёт ссылку `http://{siteId}.saas.local`.

3. **Бэкенд-логика (примеры кода, которые должны быть реализованы):**
- `DynamicDatabaseService.java` — метод `createTenantDatabase(String businessType)`:
  * подключается к админской БД
  * выполняет `CREATE DATABASE ...`
  * запускает скрипты из `db-schemas/{businessType}/schema.sql`
- `QueryTemplateService` — для каждого типа бизнеса хранит готовые запросы (например, для `MUSIC_LABEL`: `getTopTracks`, `updateProfile`, `addRelease`).
- Безопасность: каждый tenant имеет своего пользователя БД с ограниченными правами.

4. **Фронтенд, который "офигенно выглядит":**
- Анимированный переход между шагами (Framer Motion)
- Градиентный фон + Glassmorphism для панели инструментов
- Тёмная тема по умолчанию с акцентным цветом (#7C3AED)
- Адаптив под 3 размера экрана
- При загрузке — скелетоны с shimmer-эффектом
- Пример первого экрана (дашборд со списком сайтов клиента, кнопка "Создать сайт", график Recharts с активностью)

5. **Что должно работать сразу после `docker-compose up`**:
- Авторизация (заглушка, но JWT-механизм готов)
- Создание нового сайта → появление новой БД → редирект в конструктор
- Добавление блока на канвас → сохранение в Redis (черновик)
- Публикация → сборка в папке `/generated/{siteId}`
