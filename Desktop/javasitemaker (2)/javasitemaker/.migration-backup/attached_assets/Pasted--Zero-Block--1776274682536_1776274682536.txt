Необходимо реализовать переработанный Zero Block как отдельный режим редактирования блока. При выборе этого режима пользователю должен открываться пустой холст (canvas), на котором он может самостоятельно размещать элементы. В интерфейсе должна быть кнопка добавления элементов, при нажатии на которую открывается список доступных элементов: текст, изображение, кнопка, HTML-код, фигуры (прямоугольник, круг, линия), иконки и видео. После добавления элемент появляется на холсте, где пользователь может перемещать его с помощью drag & drop, изменять его размер с помощью мыши (resize), а также задавать точные координаты (x и y). Для каждого элемента должна быть доступна правая панель настроек, которая открывается при его выборе. В этой панели должны настраиваться все параметры элемента: размеры (ширина, высота), позиция, выравнивание, слои (z-index), прозрачность, фон, цвет, тени, скругления. Для текстовых элементов дополнительно должна быть полная настройка типографики: шрифт, размер, жирность, курсив, межстрочный интервал, выравнивание, цвет. Также необходимо реализовать анимации появления элементов (например, появление слева, справа, сверху, снизу или плавное появление). Все изменения должны применяться в реальном времени прямо на холсте.
Во всех блоках и элементах конструктора необходимо реализовать единую систему кастомизации. Все настройки должны быть структурированы и разделены на категории, чтобы пользователь не терялся. Обязательные категории: основные настройки (размеры, положение, отображение), типографика (все настройки текста), цвета и фон (цвета текста, фона, градиенты), кнопки (если есть кнопки — их внешний вид и поведение), пункты меню (если это навигация), анимации (появление, задержка, эффекты), отступы и размеры (margin, padding, ширина, высота). Внутри каждой категории должны быть доступны все возможные параметры, которые логически к ней относятся. Например, в типографике — все свойства текста, в кнопках — цвета, hover-состояния, скругления и т.д. Эта система должна быть одинаковой для всех блоков, чтобы пользователь один раз понял логику и мог применять её везде.
Блок Hero необходимо переименовать в более понятное название для обычного пользователя, например «Главный экран». Это изменение должно отразиться во всех местах интерфейса, где используется этот блок.
В блоке Header необходимо полностью изменить работу ссылок. Вместо ручного ввода ссылок пользователь должен выбирать страницы из списка уже созданных страниц сайта. Этот список должен подгружаться автоматически. Дополнительно каждому блоку на странице должен автоматически присваиваться уникальный ID, чтобы на него можно было ссылаться из навигации. Пользователь должен иметь возможность выбрать этот ID при настройке меню. Кнопка «Связаться» должна стать опциональной — пользователь может включить или отключить её отображение. Также для неё нужно добавить варианты поведения: переход по ссылке, открытие формы, прокрутка к определённому блоку. Настройки кнопки должны быть приведены к общему виду (как у других элементов), без неудобных выпадающих списков — использовать такой же интерфейс выбора, как и для ссылок.
Необходимо пересмотреть работу ширины блоков. В Header и Footer нельзя давать пользователю возможность вручную задавать ширину — они всегда должны занимать 100% ширины страницы. Для мобильной версии необходимо реализовать hamburger-меню, которое появляется вместо обычного меню. Для него также должны быть отдельные настройки: внешний вид, анимация открытия, положение на экране.
Нужно реализовать единый редактор текста для всего проекта. При редактировании текста должно открываться отдельное окно с полноценным функционалом форматирования. Пользователь должен иметь возможность изменять размер текста, делать его жирным или курсивным, выравнивать, задавать отступы, вставлять ссылки, создавать списки и т.д. Очень важно, чтобы можно было форматировать не весь текст целиком, а отдельные его части (по выделению, как в текстовых редакторах). Этот редактор должен использоваться во всех блоках без исключения.
Необходимо исправить существующие ошибки в блоках. В блоке «Отзывы» сейчас не отображается подзаголовок и не загружается изображение по ссылке — это нужно исправить. В блоке «Галерея» не работает изменение количества колонок — нужно починить эту логику, чтобы сетка реально менялась. Также добавить возможность отображать текст под изображениями, который редактируется через общий текстовый редактор. В блоке «Видео» сейчас не отображается описание — нужно исправить вывод этого текста. В блоке «Команда» изображения должны отображаться прямо в редакторе; если по техническим причинам это невозможно, нужно явно уведомить пользователя, что изображения будут видны только в режиме предпросмотра.
В блоке «Блог» необходимо реализовать полноценную систему статей. Каждая статья должна иметь заголовок, краткое описание, изображение и кнопку (например «Подробнее»). Пользователь должен иметь возможность задать текст этой кнопки и указать ссылку — либо на внутреннюю страницу сайта, либо на внешний ресурс. При этом структура блока должна поддерживать отображение нескольких статей.
Нужно добавить поддержку форм. Пользователь должен иметь возможность добавлять формы в разные блоки. Форма должна состоять из настраиваемых полей: текстовое поле, email, телефон, многострочный текст, чекбоксы. Для каждого поля должны быть настройки: placeholder, обязательность заполнения, валидация. Также нужно реализовать отправку данных формы — пользователь должен выбрать, куда отправлять данные (email, Telegram, Google Sheets), и эта отправка должна реально работать.
Нужно добавить блок карты. Пользователь должен иметь возможность указать адрес или координаты, выбрать тип карты (например Google или Yandex), а также вставить API-ключ. Карта должна корректно отображаться на сайте в зависимости от выбранного сервиса.
Необходимо реализовать фильтрацию блоков по тематике сайта. Когда пользователь выбирает тип сайта (например интернет-магазин или блог), ему должны показываться только те блоки, которые подходят для этой тематики. Это упростит интерфейс и уберёт лишние элементы.
Нужно добавить настройки сайта и страниц. В настройках сайта должны быть: SEO (title, description), выбор главной страницы, загрузка favicon. Для каждой страницы должны быть отдельные SEO-настройки. Также необходимо реализовать страницу ошибки (404): должна быть стандартная версия по умолчанию и возможность создать свою кастомную страницу.
Необходимо добавить базовый e-commerce функционал. Пользователь должен иметь возможность создавать товары (название, цена, изображение), отображать их на сайте и добавлять в корзину. Корзина должна позволять добавлять и удалять товары, а также считать итоговую сумму. Также нужно реализовать простой личный кабинет пользователя на создаваемом сайте (регистрация и вход). Процесс оплаты должен быть реализован как симуляция: при нажатии на кнопку «Оплатить» система случайным образом определяет результат — успешная оплата или ошибка — и показывает соответствующее уведомление пользователю.
