Тема: Создание редактора блоков, аналогичного Tilda Zero Block

Описание: Требуется разработать веб-приложение с редактором блоков в стиле WYSIWYG, который по своей функциональности максимально точно воспроизводит принципы работы "Zero Block" в конструкторе сайтов Tilda.

Точное техническое задание (ТЗ):

1. Система контейнеров (Core Layout):

Задача: Реализовать систему из двух типов контейнеров для позиционирования элементов.

Детали:

Grid Container: Центральная рабочая область с 12-колоночной сеткой. Ширина сетки должна быть фиксированной (например, 1200px) и центрироваться в окне браузера. Все элементы, помещенные сюда, по умолчанию привязаны к сетке.

Window Container: Контейнер, представляющий границы самого окна браузера (viewport). Элементы, привязанные к этому контейнеру, могут позиционироваться относительно экрана, а не сетки (например, кнопка, всегда прижатая к правому нижнему углу).

2. Свободное позиционирование и слои (Drag-and-Drop & Layers):

Задача: Обеспечить возможность абсолютного позиционирования элементов внутри контейнеров с помощью перетаскивания (drag-and-drop).

Детали:

Элементы можно свободно перемещать мышью, а их координаты (X/Y) и размеры (ширина/высота) должны задаваться как в пикселях, так и в процентах.

Реализовать панель слоев для управления Z-индексом элементов. Пользователь должен иметь возможность блокировать элемент на слое, чтобы предотвратить его случайное перемещение или редактирование.

3. Адаптивный дизайн (Responsive Design):

Задача: Сделать редактор, поддерживающий адаптивную верстку.

Детали:

Предоставить возможность переключаться между популярными разрешениями экрана (например, 320px, 480px, 768px, 1024px, 1200px и выше).

Для каждого разрешения необходимо дать пользователю возможность настраивать индивидуальные позиции, размеры и даже видимость каждого отдельного элемента.

4. Группировка объектов (Object Group):

Задача: Реализовать функциональность, при которой несколько выделенных элементов можно объединить в единый объект (группу).

Детали:

Пользователь должен иметь возможность выделить несколько элементов (например, текст, кнопку и фигуру) и объединить их в группу (через контекстное меню или сочетание клавиш Ctrl+G / Cmd+G).

Важно: Сгруппированный объект должен вести себя как единое целое. Применение анимаций, фона, рамок или ссылок должно затрагивать всю группу целиком, а не отдельные ее элементы.

5. Анимации и интерактивность:

Задача: Добавить поддержку анимаций для отдельных элементов и групп.

Детали:

Триггеры анимации: Реализовать запуск анимаций по различным событиям: при появлении блока в области видимости (viewport), при скролле, при наведении мыши (hover) и по клику (click).

Эффекты: Включить базовые эффекты, такие как плавное появление (fade-in), смещение (slide), изменение масштаба (zoom) и эффект параллакса (parallax).

6. Векторный редактор (Vector Editor):

Задача: Интегрировать простой редактор для создания и изменения векторной графики.

Детали:

Пользователь должен иметь возможность создавать векторные элементы, используя стандартные примитивы (прямоугольник, круг, звезда, линии) и инструмент "Перо" (Pen Tool) для рисования кривых Безье.

Важна возможность редактирования заливки, обводки и размера векторных фигур.

7. Модульная сетка (Modular Grid):

Задача: Дать пользователю возможность настраивать сетку, к которой привязываются элементы.

Детали:

Реализовать настройки сетки: количество колонок, ширина колонки, расстояние между колонками (gutter) и отступы по краям (margins).

Настройки сетки должны применяться либо глобально ко всем Zero-блокам в проекте, либо индивидуально к выбранному.

8. Типы контента (Widgets):

Задача: Обеспечить поддержку добавления различных типов контента через панель инструментов.

Детали:

Реализовать основные виджеты: Текст (Text), Изображение (Image), Кнопка (Button), Фигура (Shape), Видео (Video), HTML-код (Custom HTML), Форма (Form).
