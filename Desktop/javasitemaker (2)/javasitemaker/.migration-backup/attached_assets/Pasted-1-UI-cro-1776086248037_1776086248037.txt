1. Загрузка и обработка изображений профиля
Реализовать UI-компонент выбора области изображения (cropper):
Пользователь должен иметь возможность выделить область изображения перед загрузкой.
Поддержать масштабирование и перемещение области.
Итоговое изображение сохраняется уже обрезанным.
Добавить валидацию загружаемых изображений:
Проверка формата (например: JPG, PNG, WebP).
Проверка размера файла (например: max X MB).
Проверка минимального разрешения.
При ошибке загрузки:
Показывать уведомление с конкретной причиной:
неподдерживаемый формат
превышен размер
слишком маленькое разрешение
ошибка обработки
2. Кнопки в светлой теме
Для всех кнопок:
Если фон кнопки цветной → текст автоматически становится белым.
Проверить контраст (WCAG) для читаемости.
3. Цветовая схема (темы)
Светлая тема:
Осветлить текущий тёмно-фиолетовый основной цвет (primary).
Сохранить стиль, но увеличить яркость/насыщенность умеренно.
Тёмная тема:
Оставить текущую цветовую схему без изменений.
4. Обработка выгрузки/недоступности сайта
Если сайт выгружен / недоступен:
При попытке открытия:
выполнять редирект на отдельную страницу (или внешний сайт)
страница должна явно сообщать:
сайт выгружен / отключён
возможные причины
5. Tooltip / подсказки (особенно в светлой теме)
Исправить проблему невидимого текста:
текст должен быть контрастным (например тёмный текст на светлом фоне или наоборот)
Поведение:
Tooltip должен перекрывать остальные элементы (z-index)
Если tooltip выходит за границы экрана:
автоматически смещать его влево/вверх, чтобы он полностью помещался
